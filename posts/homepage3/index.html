I rebuilt my personal website from almost scratch.

Why?

  1. Vue2 is EoL and I dont want to pick up Vue3
  2. Poor SEO, content are dynamic generated after page load
  3. I want to build interactive blog post with mdx (potentially jupyter notebook)
  4. I want dark theme

How?

  1. Migrated to Next.js
  2. Use static site generation (SSG) to pregenerate all route to each blog post, that is search engine friendly
  3. Add mdx support
  4. The template I used come with tailwind, there is theme switch built-in

What did I do?

  1. Clone a template, which is quite nice https://github.com/ChangoMan/nextjs-mdx-blog
  2. Fix the caveats within the template (like yarn is more preferrable)
  3. Translate the vue pug html templates into tsx, and decide what features to drop
  4. Copy paste the scss (would be long way to switch from scss to tailwind, not sure if that will really worth it)
  5. Script to download all blog posts to the repo
  6. Set up mdx
  7. Implemented dark theme, draw the background iamge in dark theme
  8. Implemented Nextjs specific stuff, like meta data, layouts, etc
  9. Clean up blog posts to align new structure and design.
  10. Now I need to rethink the high level design, how can I present my data in a more fluent way
  11. I migrated my domain from namecheap to cloudflare, saved me a lunch each year. But I have to change my DNS as well so there were some chaos on the migration day (for example cloudflare enfore HTTPS and proxy to github page HTTP, but github page try to redirect to HTTPS which lead it back to cloudflare, the solution was to keep cloudflare to github connection secure)

What are the benefits?

  1. Dont need to maintain legacy Vue2
  2. Better SEO
  3. Cool dark theme
  4. Interactive blog post - example - set up NginX
